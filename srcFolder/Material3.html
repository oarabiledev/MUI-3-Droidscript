Material3 Color Schemes & Icons

Material Design 3 has a new way of handling colors, and how they are implemented, so as of current Droidscript cannot use dynamic theming so its not an option:

Firstly you must have your own color pallets to obtain this, go to this url and customize your apps palletes:

Material Custom Color Builder
  1. remember use the custom theme toggle.
  2. remember under ‘Extend Colors’, this will not theme your app at all,stick to core colors.

After getting your palletes done, you have to export the theme, you will export the theme as ‘Android Views’.

This will download a zip file, To setup your apps light theme follow the directory:

Downloads > material-theme > values > colors.xml

You will now have to change XML to JSON, use any converter.

For example: Best XML to JSON Converter Online (jsonformatter.org)

Make this file and place it in your app directory in the uxDesign folder that you will create for your app, save the file as appTheme.json

JSON is faster and we want speed, I could use XML, but I want it fast.

Load The Plugin First ::

app.LoadPlugin('Material3')

In your code, to initialize colors, write the following code:

app.CreateMaterial3(theme,icon,dir)
  • theme is the default theme.
  • icon is the app's icon outline, options are:
    1. Sharp As : 'sharp'
    2. Round As : 'round'
    3. Outlined As : 'outline'
    4. Two-Tone As : 'two-tone'
  • dir is the directory where appTheme.json is, if it is within uxDesign/appTheme.json write either 'default' or 'dflt' 
    
//Always Have The cfg.MUI option as 
//the plugin relies on it.
cfg.MUI
app.LoadPlugin('Material3');

function OnStart(){
 app.CreateMaterial3('dark','outline','dflt');
 lay = ui.addLayout('Linear,'FillXY');
 app.AddLayout(lay);
}

Material3 Layouts & Ds Layouts

Material3 layouts have their background by default acording to M3ColorScheme

A layout is the basic ui component of an app. It is where you add other ui components such as buttons, textfields, checkboxes, list, images and many more. To add a layout, simply call the addLayout method of the ui object.

Reference How DS Layouts Work, It Is The Same, Also All Layout Methods Are Available.

ui.addLayout(type,options) 


        
cfg.MUI
app.LoadPlugin('Material3');

function OnStart(){

app.CreateMaterial3('dark','outline','dflt');
lay = ui.addLayout('Linear,'FillXY');
app.AddLayout(lay);

Material3 Buttons & FAB Types

Elevated Buttons

To add elevated button simply invoke ::

ui.addElevatedButton(btnName, width, height, icon, parentLay)

Filled Buttons

To add a filled button simply invoke ::

ui.addFilledButton(btnName, width, height, icon, parentLay)

FilledTonal Buttons

To add a filled tonal button simply invoke ::

ui.addFilledTonalButton(btnName, width, height, icon, parentLay)

Outlined Buttons

To add an outlined button simply invoke ::

ui.addOutlinedButton(btnName, width, height, icon, parentLay)

Text Buttons

To add a text button simply invoke ::

ui.addTextButton(btnName, width, height, icon, parentLay)

FAB's

To add each FAB Type consider the following::


  1. Normal Sized FAB

    2. ui.addFAB(icon, layout)

  2. Small Sized FAB

    3. ui.addSmallFAB(icon, layout)

  3. Big FAB

    4. ui.addLargeFAB(icon, layout)

Extended FAB

To add an extendedFAB simply invoke ::

ui.addExtendedFAB(btnName, icon, width, parentLay)
